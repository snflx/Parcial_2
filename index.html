Descripción del Dataset

Este proyecto hace uso de un dataset exhaustivo (ej. AuthentiText_X_2026_AI_vs_Human) que contiene muestras de texto tanto redactadas orgánicamente por humanos como sintetizadas por potentes Modelos de Lenguaje Grando (LLMs) como GPT-4 o Claude.

Fuente de Datos: Kaggle / Dataset Público Simulado.
Objetivo: Descubrir patrones ocultos (asimetría semántica, perplejidad, legibilidad) para identificar la autoría del texto.

Diccionario de Variables Principales

Variable Tipo Descripción
author_type Categórica Variable objetivo (Target). Puede ser 'Human' o 'AI'.
content_text Texto Cadena de texto generada.
perplexity_score Numérica Medida de qué tan bien un modelo predice una muestra. Menor perplejidad suele ser IA.
burstiness_index Numérica Variación en la complejidad y longitud de oraciones a lo largo del texto. Humanos suelen tener mayor variación.
semantic_coherence_score Numérica Flujo semántico y coherencia estructural.
lexical_diversity_ratio Numérica Proporción de palabras únicas vs palabras totales.

Explicación del Proceso ETL

1. Extract (Ingesta)

Conexión automatizada al origen (`.csv` o `.xlsx`). Carga en DataFrame de pandas, verificando la sanidad e integridad del dataset crudo.

2. Transform (Transformación)

Dropear falsos nulos, remover duplicados, imputación en caso de NaN, y generación de nuevas métricas (Feature Engineering como longitud del texto).

3. Load (Carga)

Se inyecta la base limpia directamente en memoria para que el dashboard de Streamlit genere los reportes. En producción, se cargaría a un Data Warehouse.
